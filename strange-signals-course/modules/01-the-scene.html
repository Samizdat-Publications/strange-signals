258,000 pushpins, a dark map, and one very deliberate architectural choice.

Before we start

This is a six-part tour of a real codebase called Strange Signals — an interactive map of paranormal sightings. The goal isn’t to make you a programmer. It’s to give a “vibe coder” — someone who builds with AI but doesn’t read code — the vocabulary and mental map to steer a project like this with confidence.

By the end you’ll know what every file in the project is for, why the app is structured the way it is, and what tradeoffs the authors picked. We’ll move like a detective working an evidence board: pin things up, connect them with string, step back, and finally see the pattern.

The moment you open it

You open Strange Signals. A radar spins. A loading bar fills. Then 258,000 pushpins drop onto a dark map of the United States. You click a green cluster in Washington state — 3,536 UFO sightings live inside that one shape. How did a quarter of a million records from five different websites end up clickable on your laptop?

What this thing actually does

Three jobs, one screen. Everything else in the project exists to serve one of these three.

1
Plots 258K paranormal reports on a map

UFO / UAP sightings, Bigfoot encounters, and haunted places — all visible at once, filterable by year, state, and keyword.

2
Overlays real-world things nearby

Military bases, earthquakes, caves, fireballs, restricted airspace — toggleable layers to see what lines up with the sightings.

3
Hands you the steering wheel — and an AI co-pilot

An assistant named SIGNAL can drive the map for you: filter, zoom, correlate, and even write an investigation report.

This is the bar of buttons across the top of the app.

A button labeled MARKERS. The data-view tag is a little note stuck on the button so the JavaScript knows which view to switch to when it’s clicked. The word active means this one is currently selected — it’ll look highlighted.

A button labeled HEATMAP, tagged with its view name.

A button labeled HEX DENSITY, tagged hexbin (the technical name for honeycomb grids).

A button labeled CORRELATION, tagged with its view name.

The big idea: it’s a pipeline

Here is the single most important thing to understand about Strange Signals: it is not one program. It’s a pipeline of small programs that each do one job. The map you see is the last step. Everything before it happened on someone else’s computer, was saved as a file, and shipped to your browser.

Two terms worth pausing on. A JSON file is just a text file full of records — the whole dataset is one big one. And a Web Worker is a helper inside your browser that can chew through that file without freezing the page. Module 3 goes deep on the worker; for now, just know it exists.

!
Aha! The app has no backend.

There is no backend server computing anything in real time. The JSON file was pre-computed on someone’s laptop, committed to disk, and is served as a static file. Every feature — filters, correlations, even the AI assistant — runs on your computer. No server costs, no API downtime, no database. But also: no live data. Every time the datasets update, someone has to rebuild and re-publish the JSON.

That’s the tradeoff worth internalizing. Pre-computed data is fast, cheap, and offline-capable — but rigid. Live data is flexible but requires a server, a database, and someone to keep it all running. Strange Signals chose the first path. Knowing why helps you make the same call on your own projects.
