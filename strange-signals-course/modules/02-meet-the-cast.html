Meet the Cast

The writers' room behind Strange Signals — who does what, and how they pass notes.

Who's the main character?

You open the project folder. Nine JavaScript files. Eleven Python files. Three CSS files. A pile of JSON. Nobody has labeled a single one of them "start here."

Think of this codebase the way you'd think of a TV show's writers' room. Each writer has a specific job — one plots the mystery, one punches up the jokes, one keeps the continuity binder. An episode only works when they collaborate. The files here are the same: each has a role, and the show falls apart if you don't know which one to talk to.

Here's the cast list, annotated.

The six who carry the show

Out of that whole cast, six characters do the heavy lifting. Learn these names and you can point an AI coding assistant at the right file every time.

D
index.html — The Director

Loads every other script in order. Holds the nav bar, the sidebar, and the loading screen. Nothing happens until the browser reads this file.

L
strange-signals.js — The Lead

Holds all the state (allData, filteredCat, currentView), runs the map, handles every click. Everything flows through here.

P
parse-worker.js — The Data Receiver

A separate thread. Gets the raw JSON, validates it, groups records by category, ships batches back to the Lead.

H
hex-worker.js — The Math Thread

Another separate thread. Its only job: count how many sighting points fall inside each hexagon on the hex-density view. Pure math, no visuals.

A
ai-assistant.js — The Agent

Talks to Claude's API directly from the browser. Reads user messages, picks tools, executes those tools against the map.

Py
build_*.py — The Offline Prep Crew

Python scripts that run on a developer's laptop long before anything is shipped. They turn raw CSV files into the compact JSON the browser reads. They never run live.

How they pass notes: the bubble trick

Here's the architectural idea that matters most. In a writers' room, not every scribbled note on a whiteboard is for public consumption — most of it is private to one writer. A few deliberate things get taped to the door so anyone can read them.

Strange Signals does the same thing with an IIFE. The entire Lead script is wrapped in a bubble — a private room. Variables inside stay inside. At the very end of the file, the Lead picks a small set of functions and tapes them to window.StrangeSignals so the other files can reach them.

!
Aha! This is how files communicate without imports.

Modern web apps use import/export syntax to share code between files. Zero-build apps like Strange Signals skip that entirely and use the shared window object as their global namespace. Older, simpler, and every script on the page shares one giant whiteboard.
