# Impeccable — Design Context

## Project

**Strange Signals** — interactive paranormal sightings correlation map. 258K+ geocoded records (UFO/UAP, Bigfoot, Haunted Places) with spatial correlation analysis and a Claude-powered research assistant ("SIGNAL AI").

Three design surfaces exist in this repo:

1. **Main app** — `index.html`, `strange-signals.css`, companion CSS files. The working cockpit.
2. **Landing page** — `docs/index.html`. Single self-contained HTML for GitHub Pages.
3. **Course** — `strange-signals-course/`. Six-module interactive course, separate warm paper design system (`styles.css` in that folder).

**Stack is fixed:** no build tools, static HTML/CSS/JS, IIFE pattern, CDN libraries. Refine inside the existing tokens — do not rebrand.

## Users

UFO / paranormal **hobbyists at night** — enthusiasts poking around case data for fun on laptops in the evening. They arrive in a curious, mood-seeking headspace: they want the *signals* / cockpit / conspiracy feel, not a clinical dashboard. They dig in for real once hooked — so the interface must reward exploration with data density, not just vibes.

## Brand Personality

Three words: **instrumented, nocturnal, earnest**.

- **Instrumented** — this is a console, not a magazine. Readouts, tick marks, callsigns, tabular numbers.
- **Nocturnal** — designed for the 9pm-1am viewing window. Dark, low-contrast ambient, accents that glow because they're signals, not because they're decoration.
- **Earnest** — takes the subject matter seriously. Not ironic, not camp, not Halloween-store kitsch.

## Aesthetic Direction

**Lean harder into the cockpit.** More deliberate retrofuturism — instruments, callouts, typographic density, CRT patina — and make the app read as one self-consistent console rather than a dashboard with decorations stuck on top.

### Keep (tokens)

- `--bg #05060f` near-black, `--surface-solid #0b1022`
- Accents: `--green #00ff88` (primary signal), `--cyan #00d4ff` (secondary data), plus categorical `--orange --purple --pink` used sparingly
- `--font-display: Orbitron` (display / callsigns), `--font-mono: Space Mono` (body / data)
- Spacing scale `--sp-1…--sp-5`, type scale `--fs-xs…--fs-lg`
- Course surface keeps its own tokens (Bricolage / DM Sans / JetBrains Mono, warm cream) — do **not** cross-contaminate

### Anti-references — explicitly NOT

- **Cutesy paranormal** (the user's stated anti-ref). No ghost / alien / cryptid emoji as UI. No Halloween-store energy. No cartoon mascots. If a glyph is used, it's a geometric symbol (▲ ● ✦ ◆ ◈ ▣) — already the convention in the main app, which is good.
- **Hacker / Matrix cliché.** No glitch fonts, no text-rain, no excessive scanlines.
- **Corporate sci-fi AI.** No cyan-to-purple gradient text or headings, no glassmorphism-everywhere, no glowing gradient buttons.
- **Generic admin dashboard.** No border-left stripe callouts. No same-sized icon-and-text card grids repeated endlessly.

## Design Principles

1. **Instrumented, not illustrated.** Every visual element earns its place by functioning as part of the console: readouts, labels, tick marks, status lights. Decoration that doesn't function gets cut.
2. **Typographic density is the aesthetic.** Small type, tight leading, tabular numerals, monospaced labels in SCREAMING CAPS with letter-spacing. This is a feature, not a regression — it signals "instrument panel."
3. **Green is precious.** `--green` marks signal, active, alive — nothing else. If everything is green, nothing reads as signal. Most of the chrome should be `--text-dim` and `--border`.
4. **No slop fingerprints.** Hard bans: `border-left: Npx solid var(--color)` as a decorative stripe on cards/alerts/callouts; gradient text via `background-clip: text`; cyan→purple gradients; glassmorphism as decoration; emoji used as UI icons.
5. **Quiet by default, alive on interaction.** The page should feel like it's *listening* — subtle radar pulse, cursor-aware glow on a primary control — not bouncing around with entrance animations.

## Scope of "Refine, Don't Rebrand"

- Keep fonts, color tokens, layout skeleton.
- Fix type hierarchy flatness (too many sizes 11-16px with <1.15× spacing).
- Remove AI-slop patterns wherever they appear.
- Tighten the landing page so first impression matches the cockpit promise.
- Preserve course design system as-is; if it has slop, fix inside its own tokens.
